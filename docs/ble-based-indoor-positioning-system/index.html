BLE Based Indoor Positioning System

Projects • Dec 12, 2021

Overview

Indoor positioning systems enable locating the position of objects or people within buildings. There are different indoor positioning techniques available including Bluetooth Low Energy (BLE), WiFi, UWB, Near Field Communication (NFC), etc. GPS is unreliable for indoor spaces because there is no line of sight with the GPS satellites.

This project mainly focuses on an RSSI-based BLE Indoor positioning method. BLE is among the first technologies used for indoor tracking and it is widely used because of its significantly low power consumption and high adoption rate in common electronic appliances. Even though the latencies are high and data transfer capacities are low, it is not much of an issue for our purpose as data is sent periodically and only in small chunks.

Objectives

  • To configure and set up the node which is capable of collecting BLE beacons and publish that information to a MQTT topic.
  • To build RSSI based BLE fingerprint database surrounding an area of interest.
  • To classify the real-time location of a roaming user into pre-defined location areas using ML techniques.
  • To visualize the data and predictions in a user-friendly manner.

Methodology

In order to implement the system, BLE beaconing nodes and BLE sensing nodes are required. Beaconing devices periodically emit BLE beacons with their MAC address and some other encapsulated information, but there is absolutely no location intelligence built onto them. After collecting and processing information, meaningful predictions can be obtained. Sensing nodes are used to capture this emitted information along with their RSSI and origin. ESP32 which has integrated BLE and WiFi capabilities was used for this task.

During the data collection phase, RSSI data collected using ESP32 nodes had to be sent to a remote database. For this, we used an online MQTT broker. As the ESP32 has inbuilt WiFi, it could be easily connected to a WiFi hotspot and collected data were periodically published to a common MQTT topic, simultaneously by each group in JSON format along with the location IDs. At the same time a python script running on the laptop, the same MQTT topic was subscribed and all the data from every location was saved locally.  

Then the dataset was pre-processed before the training by analyzing the columns, null values, etc. Based on the nature of the dataset, we chose five classifiers such as Random Forest, SVM with linear and RBF kernels, CAT Boost as well as K-Nearest Neighbour Algorithm.          

As the visualization technique, we decided to use NodeRed platform. It has all the necessary tools that we needed such as the ability to subscribe to a MQTT topic, conditional branching and the ability to show a map.

Conclusion

Based on the above results, we can say that the highest accuracy can be obtained using Random Forest and CAT Boost classifiers. We didn't use any neural network for the prediction since there is a high chance of overfitting the data with this amount of data. Due to the class imbalance in the dataset, the performance of the classifiers is a bit lower than expected, but with a balanced dataset a good performance can be expected
